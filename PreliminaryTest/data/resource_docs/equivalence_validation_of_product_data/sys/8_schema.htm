8.2 Fundamental concepts and assumptions

This schema provides representation of inspection results of annotated model 3d equivalence. This schema is a specialization of product_data_equivalence_inspection_result_schema presented in clause 6 to the equivalence of annotated model 3d data. Therefore, all the fundamental concepts and assumptions described in 6.2 also apply.

As this schema deals with inspection result of the equivalence of annotated model 3d data, a relationship is established between annotated_model_3d_equivalence_inspection_result_representation that is a type of data_equivalence_inspection_result_representation defined in product_data_equivalence_inspection_result_schema and representation representing inspection target annotated model 3d data by annotated_model_3d_equivalence_inspected_annotated_model_3d_and_result_relationship.

NOTE 1   Geometric data newly created for representing inspection results, such as points where non-equivalence defect was detected , shall be created in the context of the inspected product data.

Annotated_model_3d_equivalence_inspection_criterion_report that is a type of data_equivalence_inspection_criterion_report defined in product_data_equivalence_inspection_result_schema represents summarized inspection result on a specific annotated model 3d equivalence criterion that includes how many entities were inspected with how many non-equivalent defects as a result. It also includes representative_inspected_value that is a typical inspected value obtained.

Annotated_model_3d_equivalence_inspection_instance_report that is a type of data_equivalence_inspection_instance_report defined in product_data_equivalence_inspection_result_schema represents the most detailed inspection results on a specific annotated model 3d equivalence criterion. An instance of this entity is created for each inspection and it includes information of inspected value and data elements on which the inspection was performed. If the exact location is required to know where an extreme inspected value was detected, instance_report_with_extreme_instances that is a type of annotated_model_3d_equivalence_inspection_instance_report that has location information on inspected data elements where non-equivalence defects were detected shall be used.

NOTE 2   The type of data element to be inspected and the type of entity to represent the location of non-equivalence defect are controlled by each subtype of annotated_model_3d_equivalence_criterion defined in annotated_model_3d_equivalence_criteria_schema. The number of annotated_model_3d_equivalence_inspection_instance_report instances to be generated can be controlled by the attributes of annotated_model_3d_criterion_with_element_report_request defined in annotated_model_3d_equivalence_criteria_schema.

Users will specify thresholds to judge if a criterion is satisfied or not, which depends on business situation. They may or may not specify the accuracy for controlling precision of numerical calculation used in the inspection. But there are many cases that the information on the accuracy that is actually applied by the inspection is useful to interpret an inspection result. For this purpose, representation of the common accuracy applied to every inspection as well as the accuracy applied to inspection of specific criterion are provided in this schema.
