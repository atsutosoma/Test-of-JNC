4.2 Fundamental concepts and assumptions

This schema provides the following information;
  • identification and definition of product data equivalence information.

    It is understood that equivalence validation information is not a constituent of a product data. In case some designed product data converted from other product data is inspected whether or not it is equivalent with the original product data , then the inspection results will be created relating to a particular couple of product data. Therefore, equivalence validation information shall be identified and defined separately from a product definition with relationship to the pertaining product definitions as necessary.

  • relationship between product data equivalence information and product data.

    This relationship is required to represent inspection results of a particular couple of product data. Actual relationship will be created between an equivalence definition entity and a couple of product definition entities.

Existence of a data equivalence definition entity implies the existence of a set of product data equivalence criteria. Related inspection results may also exist. The product data equivalence criteria are represented using entities defined in product_data_equivalence_criteria_schema and the inspection results are represented using entities defined in product_data_equivalence_inspection_result_schema.

4.3 product_data_equivalence_definition_schema type definitions

4.3.1 equivalence_product_data_select   

The equivalence_product_data_select type is a list of alternate data types. It provides a mechanism to refer to an instance of one of these data types. An equivalence_product_data_select type allows selection of either product_definition for comparing two representations in the same product_definition or product_definition_relationship for comparing different product_definitions.

EXPRESS specification:

*)
TYPE equivalence_product_data_select = SELECT
   (product_definition_or_reference,
    product_definition_relationship);
END_TYPE;
(*

4.3.2 used_equivalence_representation_select   

The used_equivalence_representation_select type is a list of alternate data types. It provides a mechanism to refer to an instance of one of these data types. An used_equivalence_representation_select type allows selection of either data_equivalence_criteria_representation or data_equivalence_inspection_result_representation.

EXPRESS specification:

*)
TYPE used_equivalence_representation_select = SELECT
   (data_equivalence_criteria_representation,
    data_equivalence_inspection_result_representation);
END_TYPE;
(*

4.4 product_data_equivalence_definition_schema entity definitions

4.4.1 data_equivalence_definition   


Warning Ent3: product_data_equivalence_definition_schema.data_equivalence_definition. Check that the description starts with " A or an data_equivalence_definition is "
An instance of data_equivalence_definition represents a set of information on product data equivalence. Identification of the product data equivalence information represented will be given by data_equivalence_definition_representation. Information on the inspected couple of product data and management data for product data equivalence shall be related with this entity as necessary.

EXPRESS specification:

*)
ENTITY data_equivalence_definition;
  description : text;
END_ENTITY;
(*

Attribute definitions:


Warning Attr1: The first word of the attribute definition should be lower case. It is: A

description: A textual description characterizing the data_equivalence_definition.

4.4.2 data_equivalence_definition_relationship   


Warning Ent3: product_data_equivalence_definition_schema.data_equivalence_definition_relationship. Check that the description starts with " A or an data_equivalence_definition_relationship is "

Warning Ent7: check for express identifier not bold nor linked

An instance of data_equivalence_definition_relationship represents a relationship between two data_equivalence_definitions.

EXAMPLE    When product data equivalence of a CAD data is checked multiple times with different requirements, data_equivalence_definition representing the former inspection can be related with the one representing the latter inspection using this entity to show the sequence of inspections.

EXPRESS specification:

*)
ENTITY data_equivalence_definition_relationship;
  description : text;
  relating_data_equivalence_definition : data_equivalence_definition;
  related_data_equivalence_definition : data_equivalence_definition;
END_ENTITY;
(*

Attribute definitions:


Warning Attr1: The first word of the attribute definition should be lower case. It is: A

description: A textual description characterizing the data_equivalence_definition_relationship.

relating_data_equivalence_definition: one data_equivalence_definition that relates to related_data_equivalence_definition.

related_data_equivalence_definition: another data_equivalence_definition that is related with relating_data_equivalence_definition.

4.4.3 data_equivalence_definition_representation_relationship   


Warning Ent3: product_data_equivalence_definition_schema.data_equivalence_definition_representation_relationship. Check that the description starts with " A or an data_equivalence_definition_representation_relationship is "
An instance of data_equivalence_definition_representation_relationship represents a relationship between an instance of data_equivalence_definition and an instance of pertaining representation of product data equivalence. The used_representation may be a data_equivalence_criteria_representation as defined in product_data_equivalence_criteria_schema or a data_equivalence_inspection_result_representation as defined in product_data_equivalence_inspection_result_schema.

EXPRESS specification:

*)
ENTITY data_equivalence_definition_representation_relationship;
  description : text;
  definition : data_equivalence_definition;
  used_representation : used_equivalence_representation_select;
END_ENTITY;
(*

Attribute definitions:


Warning Attr1: The first word of the attribute definition should be lower case. It is: A

description: A textual description characterizing the data_equivalence_definition_representation_relationship.

definition: the data_equivalence_definition whose information is represented by used_representation.

used_representation: the representation that represents the information of definition.

4.4.4 product_data_and_data_equivalence_relationship

An instance of product_data_and_data_equivalence_relationship represents a relationship between the product data equivalence information defined by data_equivalence_definition and the target product data inspected.

  data_equivalence : data_equivalence_definition;

Warning Attr1: The first word of the attribute definition should be lower case. It is: A

Warning Attr1: The first word of the attribute definition should be lower case. It is: T

product_data: The product data whose equivalence is inspected.

data_equivalence: product data equivalence information obtained by inspecting product_data.



*)
